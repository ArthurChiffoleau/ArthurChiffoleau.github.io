Tribal idle

Description

When I started working on this project at Pinpin Team, they were already advanced in the project and needed someone to balance the content of their game, so they hired me as Game Economic Designer intern. The game was inspired by "Afk Arena" with ambition to simplify the economy and put more focus on hero's placement. If you are not familiar with the genre, players choose their hero composition, positions and then battle against enemy heroes automatically, with only the possibility to choose when to use the ultimate ability of each hero. The winner is determined by who is the last standing on the battlefield, and is impacted by which heroes were chosen, if they synergize well, their level and their equipment. The last two parts can be improved during the meta loop of the game, which consist of getting rewards for completing, battle, quest, achievement and opening some chest.

What I have done

  • Balanced the 32 heroes (active ability, passive abilities, attributes)
  • Created a spreadsheet to balance current (32 heroes) and future heroes
  • Created a Python simulation to automate battle
  • Created a pipeline to import data from my spreadsheet to Unity and the live cloud service
  • Improved tools to simulate battle in editor
  • Created a unity tools to get details breakdown of past battle
  • Determined the meta game (level up price, upgrade price, items stats ...)
  • Used JavaScript automation in google spreadsheet to automate repetitive task
